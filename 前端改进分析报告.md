# CloudAlertSystem 前端改进分析报告

> 分析对象：MFCClient（MFC 图形客户端）
> 分析日期：2026-07-03

---

## 一、核心架构问题（高优先级）

### 1. 登录界面与主界面未分离 ⭐⭐⭐

**现状**
所有 UI 元素（登录控件、预警列表、编辑面板）全部塞在同一个 `CMainFrame` 窗口里。登录前功能按钮只是灰掉，但控件都可见。

**问题**

- 登录前用户就能看到所有功能，体验不专业
- 登录区和功能区混在一起，视觉层级混乱
- 没有"先登录、后使用"的仪式感

**改进方案**

- 新建 `CLoginDlg` 登录对话框（基于 `CDialog`）
- 应用启动时先弹出登录对话框
- 登录成功后关闭对话框，再显示主窗口
- 登录失败停留在登录框，密码清空

### 2. 缺少登出/用户切换功能 ⭐⭐

**现状**
只有登录，没有登出入口。

**改进方案**

- 登录后在顶部显示"当前用户：xxx"和"登出"按钮
- 登出后清空用户状态，禁用所有功能按钮
- 可选：登出后重新弹出登录对话框

### 3. 用户状态显示不醒目 ⭐⭐

**现状**
只有底部状态栏一行小字显示登录状态。

**改进方案**

- 窗口顶部标题栏下方增加用户信息栏
- 显示：用户名、连接状态（在线/离线）、服务器地址

## 二、布局与排版问题（中优先级）

### 4. 顶部区域过于拥挤 ⭐⭐⭐

**现状**
标题 + 用户名/密码/邮箱输入框 + 3个按钮（Login/Register/Set Email）全部挤在顶部 100px 高度内。

**问题**

- 按钮太小太密，容易误点
- "设置邮箱"是登录后才能用的功能，却和登录按钮平级摆放
- 注册和登录是两个流程，混在一起容易混淆

**改进方案**

| 状态  | 顶部显示内容                         |
| --- | ------------------------------ |
| 未登录 | 应用标题 + 简洁的登录表单（用户名、密码、登录/注册按钮） |
| 已登录 | 左侧：应用标题 + 当前用户名；右侧：连接状态 + 登出按钮 |

### 5. 右侧编辑面板按钮分组混乱 ⭐⭐

**现状**
5个按钮垂直堆叠：Add Price → Add Time → Modify Price → Modify Time → Delete。

**问题**

- "新增"和"修改"是两类操作，没有视觉分组
- Delete 和 Modify 放一起，误删风险高

**改进方案**

```
┌─ 新增预警 ────────┐
│  [Add Price] [Add Time] │
└───────────────────┘
┌─ 修改预警 ────────┐
│ [Modify Price] [Modify Time] │
│      [Delete] 红色     │
└───────────────────┘
```

### 6. 筛选区域位置不合理 ⭐⭐

**现状**
Status 下拉框 + Refresh 按钮放在右侧面板最底部。

**问题**
筛选是针对列表的操作，应该和列表放在一起。

**改进方案**

- 移到预警列表的顶部，作为列表工具栏
- 左侧：状态筛选下拉框；右侧：刷新按钮

### 7. 列表与编辑面板交互不直观 ⭐

**现状**
点击列表项会自动填充到右侧编辑框。

**问题**

- 用户可能不知道选中后可以编辑
- 修改后不知道要点 Modify 按钮才生效
- 没有"取消"或"重置"按钮

**改进方案**

- 添加"编辑选中项"按钮，点击后才进入编辑态
- 添加"重置"按钮，恢复选中项的原始值
- 编辑状态有视觉提示（如边框变色）

## 三、交互体验问题（中低优先级）

### 8. 缺少输入验证 ⭐⭐

**现状**
所有输入框都是纯文本，无格式校验。

**风险**

- 价格输入非数字 → 服务端报错
- 时间格式错误（如 `25:00`）→ 预警不触发
- 合约代码格式错误 → 无法匹配行情

**改进方案**

- 价格框：只允许数字和小数点
- 时间框：自动格式化为 `HH:MM:SS`
- 实时提示：输入错误时输入框边框变红 + 提示文字

### 9. 登录安全问题 ⭐⭐

**现状**

- 登录失败后密码框不清空
- 登录成功后密码框也不清空
- 密码明文保留在编辑框控件中

**改进方案**

- 登录尝试后立即清空密码框
- 可选：添加"显示密码"复选框

### 10. 操作反馈不足 ⭐

**现状**
操作结果只在底部状态栏显示一行文字。

**改进方案**

- 重要操作（添加成功、删除成功）用消息框确认
- 或用 Toast 式提示（右上角飘出，自动消失）

## 四、代码架构问题（开发维护角度）

### 11. 单文件职责过重 ⭐⭐

**现状**
`MFCClient.cpp` 一个文件塞了所有逻辑：UI 布局、网络通信、业务处理、消息循环。

**问题**

- 代码耦合度高，改一处容易影响其他
- 难以单元测试
- 新人上手困难

**改进建议**

```
MFCClient/
├── ui/
│   ├── MainFrame.h/cpp      // 主窗口
│   ├── LoginDlg.h/cpp       // 登录对话框
│   └── AlertPanel.h/cpp     // 预警编辑面板
├── net/
│   └── TcpClient.h/cpp      // 网络层（已有）
└── biz/
    └── AlertManager.h/cpp   // 预警业务逻辑
```

### 12. 硬编码配置 ⭐

**现状**
服务器 IP 和端口写死在代码里：

```cpp
#define SERVER_HOST "127.0.0.1"
#define SERVER_PORT 8888
```

**改进方案**

- 登录界面增加"服务器地址"和"端口"输入框
- 或从配置文件读取
