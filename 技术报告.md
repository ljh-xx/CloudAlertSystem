**线程模型：**

| 线程              | 创建方式                | 职责                          |
| --------------- | ------------------- | --------------------------- |
| 主线程             | 程序入口                | 初始化各模块，`Sleep(INFINITE)` 阻塞 |
| CTP线程           | CTP API内部创建         | 驱动SPI回调（行情推送、登录响应等）         |

| AlertEngine定时线程 | `CreateThread`      | 每1秒调用 `CheckTimeAlerts()`   |

## 2. 通信协议详解

本系统采用 **UTF-8文本行协议**，每条消息以 `\n` 结尾，无二进制帧头。

### 2.1 客户端 → 服务器命令

| 命令           | 格式                                   | 说明                                         |
| ------------ | ------------------------------------ | ------------------------------------------ |
| LOGIN        | `LOGIN <用户名> <邮箱>`                   | 登录/注册，同时更新邮箱                               |
| ADD_PRICE    | `ADD_PRICE <用户名> <合约> <触发价> <条件>`    | 添加价格预警（条件：0=≥，1=≤）                         |
| ADD_TIME     | `ADD_TIME <用户名> <合约> <HH:MM:SS>`     | 添加时间预警                                     |
| DELETE       | `DELETE <预警ID>`                      | 软删除预警单（设status=2）                          |
| MODIFY_PRICE | `MODIFY_PRICE <预警ID> <合约> <价格> <条件>` | 修改价格预警，同时重置为未触发                            |
| MODIFY_TIME  | `MODIFY_TIME <预警ID> <合约> <HH:MM:SS>` | 修改时间预警，同时重置为未触发                            |
| QUERY        | `QUERY <用户名> <status>`               | 查询预警列表（status: -1=全部, 0=未触发, 1=已触发, 2=已删除） |
| SET_EMAIL    | `SET_EMAIL <用户名> <邮箱>`               | 单独更新离线通知邮箱                                 |

### 2.2 服务器 → 客户端响应

| 响应        | 格式                                                   | 说明                           |
| --------- | ---------------------------------------------------- | ---------------------------- |
| OK        | `OK`                                                 | 操作成功（无附加数据）                  |
| OK + 数据   | `OK <预警ID>`                                          | ADD_PRICE/ADD_TIME 成功，返回新建ID |
| ERR       | `ERR <错误描述>`                                         | 操作失败原因                       |
| ALERT     | `ALERT <id> <type> <合约> <触发价> <条件> <时间> <状态> <创建时间>` | QUERY响应中每条预警记录               |
| END       | `END`                                                | QUERY响应结束标志                  |
| TRIGGERED | `TRIGGERED <id> <合约> <最新价> <时间戳>`                    | **服务器主动推送**，表示预警触发           |

### 2.3 协议使用示例

```
→ LOGIN alice alice@example.com
← OK

→ ADD_PRICE alice rb2609 3200.0 1
← OK 16                         ← 预警ID=16

→ ADD_TIME alice AU2608 21:30:00
← OK 17

→ QUERY alice -1
← ALERT 16 0 rb2609 3200.0000 1 - 0 2026-07-01 21:00:00
← ALERT 17 1 AU2608 0.0000 0 21:30:00 0 2026-07-01 21:00:05
← END

（当rb2609价格跌破3200时，服务器主动推送：）
← TRIGGERED 16 rb2609 3195.0000 2026-07-01 21:35:42
```

## 3. 数据库设计

数据库文件：`alerts.db`（SQLite，WAL模式，程序目录下自动创建）

### 3.1 users 表

```sql
CREATE TABLE users (
    id       INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,   -- 用户名，唯一
    email    TEXT NOT NULL DEFAULT '' -- 离线通知邮箱
);
```

### 3.2 alerts 表

```sql
CREATE TABLE alerts (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id       INTEGER NOT NULL,     -- 关联 users.id
    alert_type    INTEGER NOT NULL,     -- 0=价格预警, 1=时间预警
    contract      TEXT NOT NULL,        -- 合约代码 如 rb2609
    trigger_price REAL    DEFAULT 0,    -- 价格预警目标价（时间预警为0）
    condition     INTEGER DEFAULT 0,    -- 0=价格>=目标价, 1=价格<=目标价
    trigger_time  TEXT    DEFAULT '',   -- 时间预警 "HH:MM:SS"（价格预警为空）
    status        INTEGER DEFAULT 0,    -- 0=未触发, 1=已触发, 2=已删除
    created_at    TEXT NOT NULL,        -- 创建时间 "YYYY-MM-DD HH:MM:SS"
    triggered_at  TEXT    DEFAULT ''    -- 触发时间（未触发时为空）
);
```

**关键设计点：**

- `TriggerAlert()` 使用 `WHERE id=? AND status=0`，防止同一预警重复触发
- `status=2` 为软删除，历史记录保留不丢失
- `ModifyPriceAlert` / `ModifyTimeAlert` 会将 `status` 重置为 0（修改后重新等待触发）
- `GetPendingContracts()` 查 `DISTINCT contract WHERE status=0`，用于服务器重启后重新订阅

### 4.2 `main.cpp` — 启动入口

**位置：** `Server/main.cpp`

**启动顺序：**

```
1. AlertDB::Init()         → 打开/创建 SQLite，建表
2. Notifier()              → 加载 EmailNotify.dll，初始化邮件发送单元
3. AlertEngine::StartTimerThread() → 启动定时线程（每秒检查时间预警）
4. TcpServer::Init() + Run() → 新线程中监听 8888 端口
5. CTPMdHandler::Connect()  → 连接CTP前置，触发SPI事件循环
6. Sleep(INFINITE)          → 主线程永久阻塞（Ctrl+C退出）
```

**重要特性：** CTP连接即使失败也不会让服务器崩溃（`非fatal`），服务器仍可接收客户端连接，CTP会自动重连。

### 4.3 `AlertDB` — 数据库访问层

**位置：** `Server/AlertDB.h` + `AlertDB.cpp`

这是系统的持久化核心，所有预警数据、用户数据均通过此模块读写。

#### 核心接口一览

| 方法                                             | 功能                                      |
| ---------------------------------------------- | --------------------------------------- |
| `Init(dbPath)`                                 | 打开数据库，建表，启用WAL和外键                       |
| `AddUser(username, email)`                     | 新增用户（`INSERT OR IGNORE`），重复用户名直接返回其ID   |
| `GetUserId(username)`                          | 按用户名查ID                                 |
| `SetEmail(userId, email)`                      | 更新用户离线邮箱                                |
| `GetEmail(userId)`                             | 获取用户邮箱（用于离线通知）                          |
| `AddPriceAlert(userId, contract, price, cond)` | 插入价格预警，返回新ID                            |
| `AddTimeAlert(userId, contract, timeStr)`      | 插入时间预警，返回新ID                            |
| `DeleteAlert(alertId, userId)`                 | 软删除（status→2），带用户隔离防越权                  |
| `ModifyPriceAlert(alertId, userId, ...)`       | 修改价格预警参数，status重置为0                     |
| `ModifyTimeAlert(alertId, userId, ...)`        | 修改时间预警参数，status重置为0                     |
| `QueryAlerts(userId, status)`                  | 查某用户的预警列表（status=-1查全部）                 |
| `GetPendingPriceAlerts(contract)`              | 查某合约所有未触发的价格预警（供引擎检查）                   |
| `GetPendingTimeAlerts(timeStr)`                | 查 `HH:MM:SS` 完全匹配的未触发时间预警               |
| `TriggerAlert(alertId, triggeredAt)`           | 将预警状态改为已触发，**条件为 `status=0`，防双触发**      |
| `GetPendingContracts()`                        | 查所有未触发预警的 `DISTINCT contract`（重启后重新订阅用） |

**线程安全说明：** SQLite 编译为 `SERIALIZED` 模式（默认），多线程并发读写安全。`AlertDB` 实例在多线程中共享，由 SQLite 内部串行化保证一致性。

### 4.4 `AlertEngine` — 预警判断引擎

**位置：** `Server/AlertEngine.h` + `AlertEngine.cpp`

这是系统的业务核心，负责"什么时候该触发预警"的判断。

#### 价格预警检查 `CheckPriceAlert(contract, lastPrice)`

- 调用者：`CTPMdHandler::OnRtnDepthMarketData()`（每次行情tick回调）
- 逻辑：查DB中该合约所有未触发价格预警 → 逐条比较条件 → 满足则调用 `TriggerAlert()` + `FireAlert()`

```cpp
// 条件判断逻辑
if (rec.condition == COND_GE && lastPrice >= rec.trigger_price) fired = true;
if (rec.condition == COND_LE && lastPrice <= rec.trigger_price) fired = true;
```

#### 时间预警检查 `CheckTimeAlerts()`

- 调用者：定时线程，每1000毫秒调用一次
- 逻辑：获取当前 `HH:MM:SS` → 查DB中 `trigger_time=?` 的未触发预警 → 依次触发

#### 触发通知 `FireAlert(rec, lastPrice, timestamp)`

1. 构造 `TRIGGERED <id> <合约> <价格> <时间>\n` 字符串
2. 查询用户邮箱
3. 构造邮件标题/正文
4. 打印日志
5. 调用 `Notifier::Notify()` 分发通知

#### 动态订阅 `SubscribeContract(contract)`

- 调用者：`ClientSession::HandleAddPrice/Time()`
- 当客户端添加一个新合约的预警时，立即订阅该合约行情，无需重启

### 4.5 `TcpServer` — TCP监听服务

**位置：** `Server/TcpServer.h` + `TcpServer.cpp`

**职责：** 在独立线程中运行，持续调用 `accept()` 等待客户端接入，每接入一个客户端就创建一个新线程运行 `ClientSession`。

**关键实现细节：**

- `SO_REUSEADDR`：进程重启后可立即重绑同一端口（不用等TIME_WAIT）
- `CloseHandle(hThread)`：线程创建后立即 detach，session线程自我管理生命周期
- 每个客户端一个线程，最大并发取决于系统线程数（足够学生项目使用）

### 4.6 `ClientSession` — 客户端会话

**位置：** `Server/ClientSession.h` + `ClientSession.cpp`

每个TCP连接对应一个 `ClientSession` 实例，运行在独立线程中。

#### 命令解析流程

```
RecvLine() → 按字节读取直到\n → Dispatch(line)
                                    ├── HandleLogin()
                                    ├── HandleAddPrice()
                                    ├── HandleAddTime()
                                    ├── HandleDelete()
                                    ├── HandleModifyPrice()
                                    ├── HandleModifyTime()
                                    ├── HandleQuery()
                                    └── HandleSetEmail()
```

#### 各命令处理要点

| 命令        | 关键逻辑                                                         |
| --------- | ------------------------------------------------------------ |
| LOGIN     | `AddUser()` 注册或取已有ID，`SetEmail()` 更新邮箱，`RegisterUser()` 标记在线 |
| ADD_PRICE | 写DB后调 `engine->SubscribeContract()` 立即订阅行情                   |
| ADD_TIME  | 写DB后调 `engine->SubscribeContract()`                          |
| DELETE    | 软删除，带 `user_id` 验证防止删他人预警                                    |
| MODIFY_*  | 修改后 `status` 重置为0，相当于"重新激活"                                  |
| QUERY     | 格式：多行 `ALERT ...`，最后一行 `END`                                 |
| SET_EMAIL | 单独更新邮箱，用于客户端设置页面                                             |

**线程安全：** `Notifier::RegisterUser/UnregisterUser` 使用 `CRITICAL_SECTION` 保护在线用户Map。

### 4.7 `Notifier` — 通知分发器

**位置：** `Server/Notifier.h` + `Notifier.cpp`

这是连接"预警触发"和"用户感知"的桥梁。

#### 在线/离线判断

```
Notify(userId, msg, email, subject, body)
    │
    ├─ 在线（m_userSockets里有此userId）
    │      └─ send(sock, msg)  → TRIGGERED消息直达客户端
    │
    └─ 离线 或 send失败
           └─ SendEmail(email, subject, body)
                  └─ m_pEmailUnit->SendEmail(ems)  → EmailNotify.dll后台队列发送
```

#### 邮件集成实现

```cpp
// 启动时加载DLL（Notifier构造函数调用InitEmail()）
m_hEmailDll = LoadLibraryA("EmailNotify.dll");
FN_CreateEmailUnit fnCreate = GetProcAddress(..., "ENDLL_CreateEmailNotifyUnit");
m_pEmailUnit = fnCreate(
    "jinan20",          // 126邮箱账号
    "KGCSHVRQMZEETIKX", // 邮箱授权码（非登录密码）
    "smtp.126.com",     // SMTP服务器（端口25，无SSL）
    "jinan20@126.com",  // 发件人
    false               // 不使用SSL
);

// 发送时构造EMSend结构体
EMSend ems;
ems.dRecvEmails = toAddress.c_str();  // 收件人
ems.dSubject    = subject.c_str();
ems.dInfo       = body.c_str();
m_pEmailUnit->SendEmail(ems);  // 投入内部队列，后台线程异步发送
```

**注意：** `SendEmail` 是异步的（DLL内部有发送队列和工作线程），调用后立即返回，不阻塞业务线程。

**在线用户Map线程安全：** `CRITICAL_SECTION m_cs` 保护 `m_userSockets`，在 `RegisterUser`/`UnregisterUser`/`Notify` 中均持锁操作。

### 4.8 `CTPMdHandler` — CTP行情接入

**位置：** `Server/CTPMdHandler.h` + `CTPMdHandler.cpp`

实现 `CThostFtdcMdSpi` 接口，接入上期所/郑商所/大商所实时行情。

#### 连接流程

```
Connect() 
  → CreateFtdcMdApi("./ctpflow/")  // flow目录存储SPI状态文件
  → RegisterSpi(this)
  → RegisterFront("tcp://182.254.243.31:30011")
  → Init()                          // 启动CTP内部线程，回调开始工作

OnFrontConnected()                  // 前置连接成功
  → ReqUserLogin(brokerID="", userID="", password="")  // 公共行情无需账号

OnRspUserLogin()                    // 登录成功
  → engine->GetPendingContracts()   // 读DB中所有pending合约
  → Subscribe(contracts)            // 批量订阅

OnRtnDepthMarketData()              // 每个tick回调（高频！）
  → engine->CheckPriceAlert(InstrumentID, LastPrice)

OnFrontDisconnected(reason)         // 网络断开（CTP会自动重连，无需代码处理）
```

#### 断线重连机制

CTP API 内置自动重连：当 `OnFrontDisconnected` 触发后，CTP 自动尝试重连，重连成功后再次触发 `OnFrontConnected` → `OnRspUserLogin` → 重新批量订阅所有合约。整个过程无需业务代码干预。

**常见断线原因代码：**

- `4097`：服务端主动断开（非交易时段切换时常见）
- `8193`：心跳超时

**`Subscribe()` 安全保护：** 方法开头检查 `if (!m_pApi) return`，避免在 `Connect()` 调用前被意外调用导致空指针崩溃。

### 方案B：3人分工（含优化）

| 组员  | 负责内容                              |
| --- | --------------------------------- |

### MFC客户端开发起点

建议按如下顺序开发：

1. **新建MFC单文档/对话框工程**（推荐使用基于对话框的工程，简单直观）
2. **将 `TcpClient.h/.cpp` 复制进MFC工程**，加 `ws2_32.lib` 引用
