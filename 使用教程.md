# Cloud Alert System 使用教程

本文说明两种常用运行方式：

1. **Debug 开发版**：适合开发、调试、查看日志和复现问题。
2. **精简测试版**：适合发给其他用户体验和测试，不包含源码和调试文件。

---

## 一、Debug 开发版使用方式

### 1. 适用场景

Debug 版适合开发者自己使用，特点是文件较全，包含调试符号、测试脚本、数据库和更多依赖文件。优点是方便排查问题，缺点是体积较大，不适合作为最终发给普通用户的版本。

### 2. 所在目录

进入项目目录：

```bat
cd /d C:\Users\27477\Desktop\trae\CloudAlertSystem
```

Debug 输出目录为：

```text
C:\Users\27477\Desktop\trae\CloudAlertSystem\bin\x64\Debug
```

### 3. 编译 Debug 版

在项目根目录执行：

```bat
build.bat
```

编译成功后会看到输出目录：

```text
bin\x64\Debug
```

### 4. 启动服务端

进入 Debug 输出目录：

```bat
cd /d C:\Users\27477\Desktop\trae\CloudAlertSystem\bin\x64\Debug
```

启动服务端：

```bat
Server.exe
```

正常情况下，服务端窗口会出现类似日志：

```text
Cloud Alert System Server starting
Database initialized: alerts.db
TcpServer Listening on port 8888
```

如果 CTP 行情连接成功，还会看到 CTP 连接、登录、订阅相关日志。

服务端窗口需要一直保持打开，关闭后客户端会断开连接。

### 5. 启动图形客户端

重新打开一个命令行窗口，或直接在资源管理器中双击：

```text
C:\Users\27477\Desktop\trae\CloudAlertSystem\bin\x64\Debug\MFCClient.exe
```

也可以在命令行中执行：

```bat
cd /d C:\Users\27477\Desktop\trae\CloudAlertSystem\bin\x64\Debug
MFCClient.exe
```

### 6. 客户端基本操作流程

1. 打开 `MFCClient.exe`。
2. 在登录窗口输入用户名和密码。
3. 如果没有账号，点击注册，输入用户名、密码、确认密码和邮箱。
4. 登录成功后进入主界面。
5. 确认左上角显示当前用户和邮箱。
6. 在右侧面板添加价格预警或时间预警。
7. 点击列表中的预警可进入编辑模式。
8. 预警触发后，如果客户端在线，会弹出中文提示框。
9. 如果客户端离线，服务端会尝试发送邮件到绑定邮箱。

### 7. Debug 版常见文件说明

| 文件                    | 作用                       |
| --------------------- | ------------------------ |
| Server.exe            | 服务端程序，必须先启动。             |
| MFCClient.exe         | 图形客户端，普通使用主要运行这个。        |
| Client.exe            | 控制台客户端，主要用于开发调试协议。       |
| alerts.db             | SQLite 数据库，保存用户、邮箱和预警记录。 |
| server.conf           | 邮件配置文件。                  |
| thostmduserapi_se.dll | CTP 行情接口运行库。             |
| EmailNotify.dll       | 邮件发送运行库。                 |
| *.pdb                 | 调试符号文件，开发排查问题用。          |
| test_*.ps1            | 开发阶段测试脚本。                |

### 8. Debug 版注意事项

- 如果服务端无法启动，先确认 `Server.exe` 所在目录下有 `server.conf`、`EmailNotify.dll` 和 `thostmduserapi_se.dll`。
- 如果 CTP 报 `can not open CFlow file`，需要在当前运行目录下新建 `ctpflow` 文件夹。
- 如果当前价格显示为 `-`，可能是 CTP 行情未连接成功、合约暂无 tick，或合约代码不受当前行情源支持。
- Debug 目录中的 `alerts.db` 是开发数据库，里面可能包含本地测试账号和预警记录。

## 二、精简测试版使用方式

### 1. 适用场景

精简测试版适合发给其他用户体验。它只保留运行必要文件，不包含源码、工程文件、调试符号和测试脚本，用户解压后按说明启动即可。

### 2. 所在目录

当前整理好的精简测试版目录为：

```text
C:\Users\27477\Desktop\trae\CloudAlertSystem-Test-v0.1
```

### 3. 精简版目录内容

当前目录内容应包含：

```text
CloudAlertSystem-Test-v0.1\
  Server.exe
  MFCClient.exe
  thostmduserapi_se.dll
  EmailNotify.dll
  server.conf
  ctpflow\
  README-测试说明.txt
```

如果运行后自动生成数据库，还会出现：

```text
alerts.db
alerts.db-shm
alerts.db-wal
```

这些数据库文件是正常现象。

### 4. 启动服务端

进入精简版目录：

```bat
cd /d C:\Users\27477\Desktop\trae\CloudAlertSystem-Test-v0.1
```

启动服务端：

```bat
Server.exe
```

正常启动后，需要保持服务端窗口不要关闭。

### 5. 启动客户端

在同一个目录下双击：

```text
MFCClient.exe
```

或在命令行执行：

```bat
cd /d C:\Users\27477\Desktop\trae\CloudAlertSystem-Test-v0.1
MFCClient.exe
```

### 6. 测试用户推荐操作流程

1. 双击 `Server.exe`。
2. 等待服务端出现 `Listening on port 8888`。
3. 双击 `MFCClient.exe`。
4. 注册一个新账号。
5. 登录账号。
6. 检查主界面是否显示当前用户和邮箱。
7. 添加一个时间预警，设置为当前时间后 1 到 2 分钟，方便快速测试弹窗。
8. 添加一个价格预警，选择常用合约并设置目标价格。
9. 测试修改预警、删除预警、查看已删除预警。
10. 关闭客户端后等待预警触发，测试离线邮件。

### 7. 精简版注意事项

- `Server.exe` 必须先启动，`MFCClient.exe` 后启动。
- `ctpflow` 文件夹必须保留，否则 CTP API 可能报错：

```text
RuntimeError: can not open CFlow file
```

- `server.conf` 包含邮件配置，发给别人前确认该测试邮箱配置可以共享。
- 如果不想把本地账号和预警记录发给别人，可以删除 `alerts.db`、`alerts.db-shm`、`alerts.db-wal`，服务端下次启动会自动创建新数据库。
- 普通测试用户不需要 `Client.exe`、`.pdb`、`.lib`、源码文件或 Visual Studio 工程文件。

## 三、如何重新生成精简测试版

如果代码更新后需要重新整理测试包，可以先编译 Release 版：

```bat
cd /d C:\Users\27477\Desktop\trae\CloudAlertSystem
build.bat Release
```

编译完成后，Release 输出在：

```text
C:\Users\27477\Desktop\trae\CloudAlertSystem\bin\x64\Release
```

精简测试包至少需要复制：

```text
Server.exe
MFCClient.exe
thostmduserapi_se.dll
EmailNotify.dll
server.conf
ctpflow\
README-测试说明.txt
```

其中：

- `Server.exe` 和 `MFCClient.exe` 从 `bin\x64\Release` 复制。
- `thostmduserapi_se.dll`、`EmailNotify.dll`、`server.conf` 可从现有 Debug 或测试包目录复制。
- `ctpflow` 是文件夹，必须存在，可以为空。

## 四、常见问题

### 1. 客户端显示 Offline 怎么办？

检查服务端是否已经启动，并确认服务端窗口中有：

```text
Listening on port 8888
```

如果服务端没有启动，客户端会显示离线。

### 2. 当前价格为什么显示为 `-`？

常见原因：

- CTP 行情连接失败。
- 当前不是交易时段。
- 合约代码错误或当前行情源不支持。
- 服务端还没有收到该合约的最新 tick。

### 3. 为什么没有收到邮件？

检查以下内容：

- `server.conf` 是否存在。
- `EmailNotify.dll` 是否和 `Server.exe` 在同一目录。
- 邮箱账号、授权码、SMTP 地址是否正确。
- 用户是否已经设置邮箱。
- 客户端在线时默认走弹窗，不会优先发邮件。

### 4. 预警触发后为什么没有弹窗？

检查用户是否仍保持登录状态。服务端会根据用户 socket 判断在线状态，在线才推送弹窗；离线则发邮件。

### 5. CTP 报 `can not open CFlow file` 怎么办？

在 `Server.exe` 所在目录下新建：

```text
ctpflow
```

然后重新启动 `Server.exe`。
